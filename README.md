# Aglink (アグリンク) - 農業スタイル診断Webアプリ

<!-- ロゴのパスを仮置きしました -->

「農業への第一歩が、ここから見つかる。」

Aglinkは、農業に興味を持つ若者が、自分でも気づいていない「農業スタイル」を発見し、最適な農地情報と繋がるためのWebアプリケーションです。  
このリポジトリは、Next.js (App Router), TypeScript, Supabase, Dockerといったモダンな技術スタックで構築された、チーム開発プロジェクトの全ソースコードです。

## 💡 プロジェクトの背景と目的

「農業に興味はあるが、何から始めればいいか分からない」  
農業人口の減少という社会課題が叫ばれる一方、そんな想いを抱く20〜30代の若年層は増えています。  

本プロジェクトは、その「興味」と「最初の一歩」との間にある情報ギャップ」を埋めるために生まれました。  
ユーザーの性格やライフスタイルを分析する独自の「農業スタイル診断」を入り口に、漠然とした興味を具体的なアクションへと導きます。

---

## 🎯ターゲットユーザー


- 何から始めたらいいか分からない、農業初心者
- 20〜30代の若年層
- 副業や趣味として農業を始めたい人
- 地方移住を検討している人

## ✨ 主な機能

このアプリは、ユーザーの「発見」と「行動」をサポートする以下の機能を備えています。

### 1. 16タイプ 農業スタイル診断 (MVP)
MBTI風のフローで、ユーザーの動機・規模感・アプローチ・スタンスを分析。  
全16種類の農業タイプ（例：「週末ガーデナータイプ」「テクノロジー農家タイプ」）から最適なスタイルを提示します。

### 2. パーソナライズされた農地提案 (MVP)
診断結果に基づき、タイプに合った農地情報を推薦します。  
例: 「週末ガーデナータイプ」→ 都市近郊の市民農園、 「本格移住農家タイプ」→ 研修制度のある地方の農地。  
（MVPでは農地情報はモックデータを使用）

### 3. 診断結果の保存とマイページ (MVP)
Supabaseの認証機能と連携し、診断結果をあなたのアカウントに紐づけて保存します。  
いつでもマイページから自分の診断タイプや、提案された農地情報を振り返ることができます。

### 4. AIによるパーソナライズド提案（将来の展望）
AIが診断結果をさらに分析し、「あなたの地域では〇〇という支援制度があります」「あなたの性格に合った、今すぐ始められる栽培キットはこちらです」といった、 具体的な「次の一歩」を提案します。

### 5. ブックマーク & 農地詳細（将来の展望）
気に入った農地をブックマークしてマイページで比較したり、農地の詳細ページでさらに深い情報を得る機能も計画しています。

## 🎓 学園祭でのフィードバックループ

私たちは、このプロジェクトを「作って終わり」とは考えていません。  
まず、上記のMVP（Minimum Viable Product）を学園祭で展示し、ターゲットユーザーである同世代の学生や来場者に実際に触れてもらいます。

そこで得られた生のフィードバック（例: 「この診断面白い！」「もっとこういう情報が欲しい」）こそが最も価値のある資産です。  
このフィードバックを元に、AI機能の実装など、さらなるブラッシュアップを行っていくアジャイルな開発プロセスを採用しています。

## 🛠️ 技術スタック

このプロジェクトは、保守性と拡張性を重視し、モダンな技術スタックを採用しています。

- フロントエンド: Next.js (App Router), React, TypeScript
- UI: Tailwind CSS, shadcn/ui（コンポーネントライブラリ）
- バックエンド (BaaS): Supabase（データベース、認証、Server Actions）
- 開発環境: Docker, Docker Compose
- CI/CD: GitHub Actions

---

## 📂 アーキテクチャの特徴


- Next.js App Router: サーバーコンポーネントを積極的に活用し、高速なページ表示と優れた開発体験を実現しています。
- BaaS活用: Supabaseをバックエンドとして利用し、認証やDB操作はNext.jsのServer Actions経由で安全に実行します。
- コンポーネント設計: shadcn/uiをベースに、components/ui（汎用部品）とcomponents/domain（機能部品）に分離し、保守性の高いコンポーネント設計を採用しています。
- Dockerによる環境統一: チームメンバー全員が全く同じ環境で開発できるよう、Dockerfileとdocker-compose.yamlで開発環境を定義しています。
